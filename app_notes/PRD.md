# Markdown 笔记应用 - 产品需求文档 (PRD)

## 1. 执行摘要

### 产品概述
一款本地运行的跨平台 Markdown 笔记桌面应用，提供直观的文件管理、流畅的编辑体验和实时预览功能。

### 目标用户
- 开发者和技术写作者
- 需要本地文件管理的笔记用户
- 跨平台工作环境用户

### 核心价值
- **本地优先**：数据完全存储在本地，保护隐私
- **实时预览**：编辑与预览同步，所见即所得
- **高效管理**：支持文件夹工作区，批量管理笔记

## 3. 用户故事与验收标准

### 3.1 文件操作

#### US-001: 打开单个文件
**作为** 用户  
**我想** 通过文件选择器打开单个 Markdown 文件  
**以便** 快速编辑特定文件

**验收标准：**
- [ ] 支持通过菜单/快捷键触发文件选择器
- [ ] 仅显示 .md 和 .markdown 文件（可切换显示所有）
- [ ] 打开后显示在编辑器中
- [ ] 文件路径显示在标题栏或状态栏

#### US-002: 打开工作区文件夹
**作为** 用户  
**我想** 打开整个文件夹作为工作区  
**以便** 管理多个相关笔记

**验收标准：**
- [ ] 支持选择文件夹并加载为工作区
- [ ] 文件夹树显示在左侧边栏
- [ ] 自动识别所有 Markdown 文件
- [ ] 记住最近打开的工作区

#### US-003: 创建新文件
**作为** 用户  
**我想** 在工作区中创建新 Markdown 文件  
**以便** 添加新笔记

**验收标准：**
- [ ] 支持右键菜单创建新文件
- [ ] 支持快捷键创建（Ctrl+N）
- [ ] 弹出对话框输入文件名
- [ ] 自动添加 .md 扩展名（如未提供）
- [ ] 新文件自动在编辑器中打开

#### US-004: 删除文件
**作为** 用户  
**我想** 删除工作区中的文件  
**以便** 清理不需要的笔记

**验收标准：**
- [ ] 支持右键删除文件
- [ ] 删除前显示确认对话框
- [ ] 删除后从文件树移除
- [ ] 如果文件正在编辑，关闭编辑器标签

#### US-005: 重命名文件
**作为** 用户  
**我想** 重命名工作区中的文件  
**以便** 更好地组织笔记

**验收标准：**
- [ ] 支持右键重命名
- [ ] 文件名变为可编辑状态
- [ ] 重命名后更新文件树和编辑器标签

### 3.2 编辑功能

#### US-006: Markdown 编辑
**作为** 用户  
**我想** 在编辑器中编写 Markdown 语法  
**以便** 创建格式化的笔记

**验收标准：**
- [ ] 支持标准 Markdown 语法
- [ ] 支持语法高亮
- [ ] 显示行号
- [ ] 支持 Tab 缩进（2个空格）
- [ ] 支持撤销/重做（Ctrl+Z / Ctrl+Y）

#### US-007: 实时预览
**作为** 用户  
**我想** 在编辑时实时预览渲染效果  
**以便** 即时查看格式化结果

**验收标准：**
- [ ] 左右分屏布局（编辑在左，预览在右）
- [ ] 输入后 < 100ms 更新预览
- [ ] 预览支持滚动同步（可选）
- [ ] 预览样式美观，支持代码高亮

#### US-008: 切换预览
**作为** 用户  
**我想** 关闭或重新打开预览面板  
**以便** 根据需要调整工作区

**验收标准：**
- [ ] 支持按钮/快捷键切换预览显示
- [ ] 预览关闭时编辑器占满宽度
- [ ] 预览状态在会话间保持

#### US-009: 自动保存
**作为** 用户  
**我想** 文件自动保存  
**以便** 不丢失编辑内容

**验收标准：**
- [ ] 停止输入 2 秒后自动保存
- [ ] 保存时显示状态指示
- [ ] 支持手动保存快捷键（Ctrl+S）
- [ ] 保存失败时提示用户

### 3.3 界面功能

#### US-010: 文件树导航
**作为** 用户  
**我想** 在文件树中浏览和选择文件  
**以便** 快速切换编辑的文件

**验收标准：**
- [ ] 树形结构显示文件夹和文件
- [ ] 点击文件在编辑器中打开
- [ ] 支持展开/折叠文件夹
- [ ] 当前编辑文件高亮显示
- [ ] 文件按字母顺序排序

#### US-011: 多标签编辑
**作为** 用户  
**我想** 同时打开多个文件标签  
**以便** 在文件间快速切换

**验收标准：**
- [ ] 支持多个文件同时打开
- [ ] 标签显示文件名和关闭按钮
- [ ] 未保存文件显示指示标记
- [ ] 支持快捷键切换标签（Ctrl+Tab）

## 4. 功能需求

### 4.1 支持的 Markdown 语法

| 语法元素 | 优先级 | 示例 |
|----------|--------|------|
| 标题 | P0 | `# 一级标题` |
| 粗体/斜体 | P0 | `**粗体** *斜体*` |
| 列表 | P0 | `- 项目` / `1. 项目` |
| 链接 | P0 | `[链接](url)` |
| 图片 | P0 | `![描述](路径)` |
| 代码块 | P0 | \`\`\`code\`\`\` |
| 引用 | P0 | `> 引用` |
| 表格 | P1 | `\|列\|列\|` |
| 任务列表 | P1 | `- [ ] 任务` |
| 分隔线 | P1 | `---` |

### 4.2 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+O | 打开文件 |
| Ctrl+Shift+O | 打开文件夹 |
| Ctrl+N | 新建文件 |
| Ctrl+S | 保存文件 |
| Ctrl+W | 关闭当前标签 |
| Ctrl+Tab | 切换至下一标签 |
| Ctrl+Shift+Tab | 切换至上一标签 |
| Ctrl+\ | 切换预览显示 |
| Ctrl+F | 查找 |
| Ctrl+=/- | 放大/缩小字体 |

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 应用启动时间 | < 3 秒 | 冷启动测试 |
| 文件打开时间（< 1MB） | < 1 秒 | 基准测试 |
| 预览更新延迟 | < 100ms | 输入测试 |
| 内存占用（空闲） | < 200MB | 监控工具 |
| 内存占用（大文件） | < 500MB | 10MB 文件测试 |

### 5.2 兼容性需求

- **Windows**: Windows 10 及更高版本
- **macOS**: macOS 10.14 (Mojave) 及更高版本
- **Linux**: Ubuntu 20.04+ / Fedora 34+ / Arch

### 5.3 用户体验需求

- 界面语言：简体中文
- 支持暗色/亮色主题
- 可调整的分屏比例
- 流畅的动画效果

### 5.4 安全需求

- 文件访问遵循操作系统权限
- 不收集用户数据
- 无网络通信（可选更新检查除外）

## 6. 界面需求

### 6.1 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  菜单栏 (文件、编辑、视图、帮助)                              │
├──────────┬────────────────────────────┬─────────────────────┤
│          │                            │                     │
│          │                            │                     │
│  文件树   │        编辑器              │      预览          │
│          │                            │                     │
│          │                            │                     │
│  (侧边栏) │      (可调整宽度)          │    (可关闭)        │
│          │                            │                     │
│          │                            │                     │
├──────────┴────────────────────────────┴─────────────────────┤
│  状态栏 (当前文件、字数统计、保存状态)                        │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 主要界面组件

1. **侧边栏**
   - 文件树视图
   - 文件夹展开/折叠按钮
   - 右键上下文菜单

2. **编辑器区域**
   - 标签栏
   - 代码编辑器
   - 行号
   - 语法高亮

3. **预览区域**
   - 渲染后的 Markdown
   - 代码高亮
   - 滚动条

4. **状态栏**
   - 当前文件路径
   - 字数/字符数统计
   - 保存状态指示
   - 光标位置（行:列）

## 7. 平台需求

### 7.1 技术栈

- **后端**: Rust
- **前端**: Flutter Desktop
- **通信**: flutter_rust_bridge
- **状态管理**: Riverpod

### 7.2 依赖库

**Flutter:**
- `flutter_code_editor` - Markdown 编辑器
- `flutter_markdown` - 预览渲染
- `file_picker` - 文件选择器
- `riverpod` - 状态管理
- `window_manager` - 窗口管理

**Rust:**
- `flutter_rust_bridge` - FFI 绑定
- `tokio` - 异步运行时
- `notify` - 文件监控
- `pulldown-cmark` - Markdown 解析

## 8. 依赖与限制

### 8.1 外部依赖

- Flutter SDK 3.16+
- Rust toolchain 1.75+
- 各平台编译工具链

### 8.2 技术限制

- 大文件（> 10MB）可能性能受限
- 图片预览仅限于本地路径
- 不支持实时协作编辑
